# Chaves Nightmares

This is a website for horrible game, built with Next.js. It includes game information, download links, articles, blogs, and more, with internationalization support for English, Spanish, and Portuguese.

## Features

- **Game Information**: Details about Chaves Nightmares, including story, features, and gameplay tips.
- **Downloads**: Links to download the game for Android, iOS, and other platforms.
- **Articles**: In-depth articles about the game, versions, and related topics.
- **Blog**: Updates and news about the game.
- **Popular Games**: Information on other popular games like PokePath and Six Sided Streets.
- **Internationalization**: Supports multiple languages (English, Spanish, Portuguese).
- **Responsive Design**: Optimized for mobile and desktop.

## Tech Stack

- **Framework**: Next.js 14 with App Router
- **Language**: TypeScript
- **Styling**: CSS Modules / Tailwind CSS (based on globals.css)
- **Internationalization**: next-intl
- **Linting**: ESLint
- **Deployment**: Vercel

## Getting Started

### Prerequisites

- Node.js (version 18 or higher)
- npm, yarn, pnpm, or bun

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/your-username/chavesnightmares.git
   cd chavesnightmares
   ```

2. Install dependencies:
   ```bash
   npm install
   # or
   yarn install
   # or
   pnpm install
   # or
   bun install
   ```

3. Run the development server:
   ```bash
   npm run dev
   # or
   yarn dev
   # or
   pnpm dev
   # or
   bun dev
   ```

4. Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## Project Structure

- `src/app/`: Next.js app directory with pages and layouts
- `src/components/`: Reusable React components
- `src/lib/`: Utility functions and configurations
- `src/messages/`: Internationalization message files
- `public/`: Static assets like images and icons
- `plans/`: SEO audit and improvement plans

## Scripts

- `npm run dev`: Start the development server
- `npm run build`: Build the project for production
- `npm run start`: Start the production server
- `npm run lint`: Run ESLint
